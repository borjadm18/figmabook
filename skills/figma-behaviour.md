# figma-behaviour

Reads Figma prototype connections and generates custom React hooks. Updates existing component files with hook usage, adds interaction stories, and adds hook unit tests.

**Trigger:** Invoked by figma-to-storybook orchestrator when phase=BEHAVIOUR.
**Input:** `.figma-migration.json` with `progress.organisms.done[]` fully populated (all components generated).
**Output:** `use{ComponentName}.ts` hooks, updated `{ComponentName}.tsx` imports, updated stories, updated tests.

---

## Limitations (display to developer before starting)

Print this message before any work begins:
```
figma-behaviour: analysing prototype connections...

Known limitations:
  ⚠ Smart Animate transitions → comment only (no CSS animation or Framer Motion code)
  ⚠ Cross-page navigation → TODO comment only (no router integration)
  ⚠ Drag interactions → scaffolded hook, empty handlers (manual implementation required)
  ⚠ Business logic (API calls, validation, side effects) → always requires manual completion
  ~70% of each hook is generated automatically.
```

## Data extraction

### MCP mode (`config.mcp_mode: true`)
Call `mcp__figma__get_design_context` for each component across all done[] lists (atoms + molecules + organisms combined). Extract interaction hints from the response. These hints describe trigger-action pairs.

### REST mode (`config.mcp_mode: false`)
Read `interactions[]` from each component's manifest entry (populated during EXTRACT if present).

Interactions format:
```json
{
  "trigger": { "type": "ON_CLICK" | "ON_HOVER" | "AFTER_TIMEOUT" | "ON_DRAG" },
  "action": {
    "type": "OVERLAY" | "NAVIGATE" | "SWAP" | "SCROLL_TO",
    "destinationId": "{nodeId of target}",
    "navigation": "OVERLAY" | "PUSH" | "REPLACE",
    "duration": 300,
    "easingType": "EASE_IN_OUT"
  }
}
```

**Skip rule:** If a component has no `interactions[]` and no MCP interaction hints, skip it entirely. Do not add it to `progress.behaviours.done[]`.

## Trigger → hook mapping

| Trigger | Action | Hook type | Output |
|---|---|---|---|
| `ON_CLICK` | `OVERLAY` (destinationId matches component with `semanticType: overlay`) | `use{ComponentName}` with `isOpen` | New hook file + updated TSX + interaction story + hook test |
| `ON_CLICK` | `NAVIGATE` (destinationId is another component on same page) | `use{ComponentName}` with `activeIndex` | New hook file + updated TSX + interaction story + hook test |
| `ON_CLICK` | `NAVIGATE` (destinationId is a top-level page frame) | No hook | TSX gets `// TODO: router.push('{pageName}')` comment in onClick handler |
| `ON_HOVER` | `SWAP` (variant change) | No hook | TSX gets `group` class on parent + `group-hover:{variant-classes}` on element |
| `AFTER_TIMEOUT` | Any | `useAutoAdvance` | New hook file + updated TSX + interaction story + hook test |
| `ON_DRAG` | Any | `use{ComponentName}Drag` scaffold | New hook file (empty handlers) + updated TSX — mark as `"todoOnly": true` in progress |

## Hook file templates

### isOpen pattern (ON_CLICK + OVERLAY)

```ts
// use{ComponentName}.ts
// Auto-generated by figma-behaviour
// Figma prototype: ON_CLICK → OVERLAY → {destinationId} ({DestinationComponentName})
import { useState, useCallback } from 'react';

// TODO (figma-behaviour): Add any business logic, API calls, or side effects
// this interaction requires (e.g. fetch data on open, log analytics, etc.)

export function use{ComponentName}(initialOpen = false) {
  const [isOpen, setIsOpen] = useState(initialOpen);

  const open   = useCallback(() => setIsOpen(true),        []);
  const close  = useCallback(() => setIsOpen(false),       []);
  const toggle = useCallback(() => setIsOpen(v => !v), []);

  return { isOpen, open, close, toggle };
}
```

### activeIndex pattern (ON_CLICK + NAVIGATE same page)

```ts
// use{ComponentName}.ts
// Auto-generated by figma-behaviour
// Figma prototype: ON_CLICK → NAVIGATE → {destinationId} (same-page navigation)
import { useState, useCallback } from 'react';

// TODO (figma-behaviour): Define total item count or pass as parameter if dynamic.

export function use{ComponentName}(totalItems: number, initialIndex = 0) {
  const [activeIndex, setActiveIndex] = useState(initialIndex);

  const goTo = useCallback((index: number) => {
    setActiveIndex(Math.max(0, Math.min(index, totalItems - 1)));
  }, [totalItems]);

  const next = useCallback(() => {
    setActiveIndex(i => Math.min(i + 1, totalItems - 1));
  }, [totalItems]);

  const prev = useCallback(() => {
    setActiveIndex(i => Math.max(i - 1, 0));
  }, []);

  return { activeIndex, goTo, next, prev };
}
```

### useAutoAdvance pattern (AFTER_TIMEOUT)

```ts
// use{ComponentName}.ts
// Auto-generated by figma-behaviour
// Figma prototype: AFTER_TIMEOUT ({action.duration}ms) → {action.type}
import { useState, useEffect, useCallback } from 'react';

// TODO (figma-behaviour): Verify interval duration matches design intent.
// Figma duration: {action.duration}ms

export function use{ComponentName}(
  totalItems: number,
  intervalMs = {action.duration},
  autoStart = true
) {
  const [activeIndex, setActiveIndex] = useState(0);
  const [running, setRunning] = useState(autoStart);

  useEffect(() => {
    if (!running) return;
    const id = setTimeout(() => {
      setActiveIndex(i => (i + 1) % totalItems);
    }, intervalMs);
    return () => clearTimeout(id);
  }, [activeIndex, running, totalItems, intervalMs]);

  const pause  = useCallback(() => setRunning(false), []);
  const resume = useCallback(() => setRunning(true),  []);
  const reset  = useCallback(() => setActiveIndex(0), []);

  return { activeIndex, running, pause, resume, reset };
}
```

## TSX component updates

### For isOpen / activeIndex patterns: add import + hook call

```tsx
// Add at top of file:
import { use{ComponentName} } from './use{ComponentName}';

// Add inside component function, before return:
const { isOpen, open, close } = use{ComponentName}();

// Update trigger element:
<button type="button" onClick={open} ...>

// Pass state to overlay component:
<{DestinationComponent} isOpen={isOpen} onClose={close} />
```

### For cross-page navigation (ON_CLICK + NAVIGATE to page frame):

```tsx
// Add onClick to trigger element:
<button
  type="button"
  onClick={() => {
    // TODO: router.push('/{pageName}')
    // figma-behaviour: ON_CLICK → NAVIGATE → {pageFrameName}
  }}
  ...>
```

### For ON_HOVER + SWAP:

Add `group` class to the parent container element. Add `group-hover:{variant-class}` to the target element. No hook file is created.

Example: if hovering on a card changes its background from `bg-surface` to `bg-primary`:
```tsx
// Parent gets 'group':
<div className={cn('group', ...)}>
  {/* Target element gets group-hover: */}
  <div className={cn('bg-surface group-hover:bg-primary', ...)}>
```

## Story update (add interaction story)

Append to the existing `.stories.tsx` file:

```tsx
// Add import at top if not present:
import { use{ComponentName} } from './use{ComponentName}';

// Add story after existing stories:
export const WithInteraction: Story = {
  render: () => {
    // eslint-disable-next-line react-hooks/rules-of-hooks
    const { isOpen, open, close } = use{ComponentName}();
    return (
      <>
        <{ComponentName} onClick={open} />
        <{DestinationComponent} isOpen={isOpen} onClose={close} />
      </>
    );
  },
  parameters: {
    docs: {
      description: {
        story: 'Figma prototype: {trigger.type} → {action.type} → {destinationId}',
      },
    },
  },
};
```

## Test update (add hook unit tests)

Append to the existing `.test.tsx` file:

```tsx
import { renderHook, act } from '@testing-library/react';
import { use{ComponentName} } from './use{ComponentName}';

describe('use{ComponentName}', () => {
  it('starts closed', () => {
    const { result } = renderHook(() => use{ComponentName}());
    expect(result.current.isOpen).toBe(false);
  });

  it('opens on open()', () => {
    const { result } = renderHook(() => use{ComponentName}());
    act(() => { result.current.open(); });
    expect(result.current.isOpen).toBe(true);
  });

  it('closes on close()', () => {
    const { result } = renderHook(() => use{ComponentName}(true));
    act(() => { result.current.close(); });
    expect(result.current.isOpen).toBe(false);
  });

  it('toggles on toggle()', () => {
    const { result } = renderHook(() => use{ComponentName}());
    act(() => { result.current.toggle(); });
    expect(result.current.isOpen).toBe(true);
    act(() => { result.current.toggle(); });
    expect(result.current.isOpen).toBe(false);
  });
});
```

## Progress tracking

After processing each component:
- Components with generated hooks: add to `progress.behaviours.done[]` as `{ "name": "{ComponentName}", "todoOnly": false }`
- Components with TODO-only changes (cross-page nav, drag scaffold): add as `{ "name": "{ComponentName}", "todoOnly": true }`
- Components with no interactions: do NOT add to `progress.behaviours.done[]`

Update `.figma-migration.json` after each component.
