What it does

One command. A complete component library.

Run /figma-to-storybook in any project. The skill suite handles the rest.

✓ Does
  • Extracts every component, token, and page frame from your Figma file
  • Generates React + TypeScript + Tailwind CSS components
  • Creates Storybook stories with live Figma design links
  • Writes Vitest tests for every component and variant
  • Generates custom hooks from Figma prototype interactions
  • Composes full-page Storybook stories (desktop, tablet, mobile)
  • Verifies fidelity and auto-fixes common issues
  • Produces a migration report with fidelity scores per component
  • Resumable — picks up exactly where it left off
✗ Doesn't
  • Generate business logic (API calls, validation, side effects)
  • Set up routing or page navigation
  • Translate Figma Smart Animate transitions to CSS animations
  • Implement drag interactions (scaffolds them with TODOs)
  • Generate backend code
  • Replace design decisions — it mirrors what's in Figma
  • Support css-modules or styled-components (Tailwind only)

How it works

7 skills. One pipeline.

Each skill has a single responsibility. They share state via a local JSON file and run in sequence.

🔐
figma-extract

Extract & Classify

Authenticates with Figma (MCP OAuth or Personal Access Token), extracts all components, tokens, and page frames. Classifies them into atoms, molecules, and organisms. You confirm the classification before anything is generated.

🎨
figma-tokens

Design Tokens

Converts Figma variables (colors, spacing, typography, shadows) into a tailwind.config.ts theme extension and a src/tokens/index.ts file of named JS constants. Also creates the cn() utility.

⚛️
figma-component

Component Generation

Generates TSX components with Tailwind classes translated directly from Figma manifest values. Every CSS property traces to a manifest field — or is marked NOT IN FIGMA. Runs three times: atoms → molecules → organisms.

🔍
figma-verify

Fidelity Verification

Checks every generated Tailwind class against the manifest. Auto-fixes simple issues (wrong color, missing gap). Escalates complex issues (blend modes, layout changes) to you. Blocks advancement until all critical issues are resolved. Target: ≥98% fidelity.

🎣
figma-behaviour

Interaction Hooks

Reads Figma prototype connections (click triggers, timeouts, hover swaps) and generates custom React hooks. ON_CLICK → OVERLAY becomes a useModal hook. AFTER_TIMEOUT becomes useAutoAdvance. Adds interaction stories and unit tests.

📄
figma-pages

Page Composition

Takes top-level Figma page frames and assembles organisms into Storybook Page stories. Respects Y-order from Figma, applies real content values, and generates Desktop / Tablet / Mobile viewport stories.

📊
figma-to-storybook

Orchestrator + Report

Sequences all skills, manages the shared state file, handles errors with retry/skip/abort, and generates the final migration report with fidelity scores. Also runs a learning system that captures recurring fixes and proposes new rules across projects.

Output

What gets generated.

A fully structured project ready to run in Storybook.

    atoms/     Button, Input, Tag, Link

Usage

How to run a migration.

Open Claude Code in any project directory and follow the prompts.

1

Install the skills

Run the install script once. All 7 skills are copied to ~/.claude/skills/ and available globally.

2

Open Claude Code in your project

Navigate to the directory where you want to generate the component library. Can be an empty directory or an existing project.

3

Run /figma-to-storybook

The orchestrator starts. It asks for your Figma file URL (or file ID), your preferred language (TypeScript recommended), and test framework (Vitest recommended).

4

Confirm the classification

Claude shows you a table of all detected components — which are atoms, molecules, organisms — and the detected page frames. You approve or adjust before any files are generated.

5

Review fidelity checks

After each layer, Claude runs a fidelity check. Simple issues are auto-fixed. Complex issues (blend modes, layout changes) are shown to you with a description of what needs manual attention.

6

Resume anytime

If the session stops, just run /figma-to-storybook again in the same directory. It reads .figma-migration.json and continues from exactly where it left off.

Limitations

What to expect from the hooks skill.

The behaviour skill generates ~70% of each hook. Some things Figma can't communicate.

Business logic

Hooks have the correct structure (useState, useCallback, open/close/toggle) but API calls, form validation, and side effects always need manual completion. Every generated hook includes a // TODO (figma-behaviour): comment where logic is needed.

Smart Animate transitions

Figma's Smart Animate information (duration, easing, animated properties) is captured as a comment in the code but no CSS animation or Framer Motion code is generated.

Cross-page navigation

When a Figma prototype links to a different page frame, the skill inserts a // TODO: router.push('{pageName}') comment. No router integration is set up automatically.

Drag interactions

Drag interactions get a scaffolded hook with empty onDragStart, onDrag, and onDragEnd handlers. The implementation is manual.
